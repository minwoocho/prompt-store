_**User**_

AI에게 프로젝트를 생성하기 위한 스크립트를 생성하는 기능이며, 생성되는 스크립트의 목록은 아래와 같다.
@PRD.md @PROGRESS.md @.cursorrules 
@CODE_STYLE.md 
프롬프트 생성은 첨부된 이미지와 같이 프로젝트 설명, 플랫폼 선택, 프레임워크 선택, 빌더 선택, 패키지 선택, 프롬프트 생성의 순서로 진행된다.
플랫폼은 웹, 모바일, 서버 등으로 구성되며, 프레임워크, 빌더, 패키지 역시 이전 단계에 선택된 것에 종속적인 것들로만 구성되어야한다. 최종적으로 생성된 프롬프트들은 각 단계별로 선택한 것들을 반영한 프롬프트여야만 한다.

   - `.cursorrules`: 기존 규칙 내용 표시
   - `PRD.md`: 제품 비전, 목표, 기술 스택 정보 포함
   - `PROGRESS.md`: 프로젝트 진행 상황, 마일스톤, 스프린트 정보 포함
   - `CODE_STYLE.md`: 코드 작성 규칙, 컴포넌트 구조, 패키지 사용 규칙 포함

3. 추가 기능:
   - 복사 버튼: 각 문서의 내용을 클립보드에 복사할 수 있는 기능
   - 모노스페이스 폰트: 코드와 문서를 보기 좋게 표시
   - 반응형 탭: 화면 크기에 따라 스크롤 가능한 탭 인터페이스

이제 `/rules` 페이지에서 각 규칙의 상세 정보를 볼 때 네 가지 문서를 탭으로 구분하여 볼 수 있습니다. 각 문서는 해당 규칙의 플랫폼, 프레임워크, 빌더, 패키지 정보를 포함하여 자동으로 생성됩니다.
